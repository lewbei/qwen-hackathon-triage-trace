# TriageTrace — Competition Strategy (Qwen Cloud Global AI Hackathon)

> Research snapshot: 2026-07-17. Sources cited inline.

## 1. What the judges care about

The Qwen Cloud Global AI Hackathon Series is judged on four main axes (Devpost rules, 2026 edition):

| Criterion | Weight | What it means for us |
|---|---|---|
| **Technical Depth & Engineering** | 30% | Sophisticated use of Qwen Cloud APIs, custom components, algorithmic innovation, performance optimization |
| **Innovation & AI Creativity** | 30% | Novel architecture, modularity, scalability, clean code |
| **Problem Value & Impact** | 25% | Real-world pain point, productization / open-source potential |
| **Execution & Demo** | implied | Working backend on Alibaba Cloud, 3-min public video, architecture diagram |

Track 1: **MemoryAgent** asks for:

> "Build an Agent with persistent memory that autonomously accumulates experience, remembers user preferences, and makes increasingly accurate decisions across multi-turn, cross-session interactions. Participants should focus on: **efficient memory storage and retrieval, timely forgetting of outdated information, and recalling critical memories within limited context windows.**" ([Devpost official page](https://qwencloud-hackathon.devpost.com))

That maps almost one-to-one to TriageTrace's core mechanics:

| Track requirement | TriageTrace feature |
|---|---|
| Persistent memory | `MemoryRecord` + pgvector + Alembic |
| Accumulate experience | approved/rejected proposals write `procedure` / `preference` memories |
| Timely forgetting | TTL-based `expires_at`, supersession, `quarantined` status |
| Recalling within limited context | MMR + 800-token budget + utility-weighted packing |

## 2. The competitive landscape

### Direct MemoryAgent entrants

- **Raven-Memory** — a LinkedIn-posted MemoryAgent submission from the same Qwen Cloud Hackathon. It frames itself as a "deterministic memory substrate" that retrieves through "reasoned affinity" instead of pure vector similarity, with a forensic-engineering angle. It is general-purpose, not domain-specific. ([LinkedIn / Anna Tchijova](https://www.linkedin.com/posts/anna-tchijova_qwenhackathon-memoryagents-forensicengineering-activity-7467196629993807872-k4M6))
  - *Our edge*: Raven-Memory is abstract; TriageTrace is applied to a high-stakes ops domain with a **security/firewall** lens.

- Generic personal-assistant / cross-session chat memories likely dominate the track. These are easier to demo but harder to prove value.

### Adjacent projects outside the hackathon

- **Incident-response agents with RAG** (e.g., Augmentcode guide): retrieve runbooks and incident reports. They treat memory as read-only context, not as a learned, approved-and-simulated, adversarially-hardened memory layer. ([Augmentcode guide](https://www.augmentcode.com/guides/ai-agents-incident-management))
- **Temporal AI Agents / Kore.ai**: discuss time-aware memory and RAG-memory hybridity, but do not implement a validation gate or poison-quarantine for memory writes. ([Cobus Greyling, Medium 2025](https://cobusgreyling.medium.com/temporal-ai-agents-311d950381c1); [Kore.ai blog](https://www.kore.ai/blog/what-is-ai-agent-memory))
- **Memory poisoning research**: `AGENTPOISON`, `MINJA`, and the Coalition for Secure AI's *AI Incident Response Framework v1.0* describe the attack but do not ship a defensive memory pipeline. ([arXiv:2605.15338](https://arxiv.org/html/2605.15338v2); [Coalition for Secure AI PDF](https://www.coalitionforsecureai.org/wp-content/uploads/2026/03/AI-Incident-Response-1.pdf))

### What this means

The MemoryAgent track is probably crowded with chat/memory demos. A **production-ops + memory-security** angle is both rarer and more defensible: it gives us a clear problem, real evaluation data, and a story about why memory safety matters.

## 3. Why TriageTrace can win

### Strongest angles

1. **Security-first memory design** — the "memory firewall" concept directly addresses a 2025-2026 research hot topic (memory poisoning). We do not just retrieve memories; we validate, quarantine, supersede, and pack them under budget. This is a defensible, novel contribution.
2. **Track alignment** — every judging bullet (storage, retrieval, forgetting, limited context, cross-session accuracy) is implemented and measured.
3. **End-to-end Qwen Cloud integration** — `qwen3.7-plus` reasoning, `text-embedding-v4` vectors, and `qwen3-rerank` relevance scoring, with embedding cosine and deterministic BM25 fallbacks.
4. **Evaluation harness with live numbers** — `evaluations/latest.json` is committed, generated by live Qwen calls. Stateless 23.1% → Memory 84.6% accuracy; 100% policy compliance; temporal conflict resolved by newer runbook; poisoned instruction safely declined. Judges can reproduce.
5. **Production scaffolding** — FastAPI, SQLAlchemy, Alembic, pgvector, Docker, React dashboard, Alibaba Cloud deployment docs. This signals engineering seriousness.

### Current weaknesses vs. winning standard

| Weakness | Risk | Fix priority |
|---|---|---|
| No public 3-minute demo video | submission is ineligible | P0: script + record + upload |
| Live demo is best-effort; some networks may block the ECS IP | judges cannot verify | P0: add mobile-hotspot / alternate-network guidance |
| MemoryAgentBench pilot is only 3 samples / 15 questions | evaluation depth | P1: run more samples if quota allows, or label as pilot |
| Latency per run ~26–31 s | demo feels slow | P1: add async streaming + status endpoint, cache fixtures |
| UI does not surface provenance / trust score | missed security story | P1: add provenance column + trust badge |
| `RunRecord` audit trail exists but is not surfaced as an immutable audit log | weak on security depth | P1: expose `GET /api/agent/runs/{id}/events` in the UI |

## 5. Key talking points for the submission / video

- **Not another chat memory demo** — TriageTrace remembers operator-approved, simulator-screened incident outcomes in a regulated, high-stakes environment.
- **Memory safety by design** — credentials redacted, provenance scored, policies packed first, poisoned memories quarantined, stale memories superseded.
- **Measured improvement** — live Qwen evaluation shows memory mode beats the identical stateless Qwen baseline on accuracy while remaining 100% policy-compliant.
- **Built for production** — FastAPI, SQLAlchemy migrations, pgvector, Docker, Alibaba Cloud ECS with local PostgreSQL/pgvector, React dashboard.

2. `README.md` + `docs/architecture.png`.

## 7. Conclusion

TriageTrace is a strong fit for the MemoryAgent track because it turns the abstract "persistent memory" requirement into a concrete, safety-critical application. The **memory firewall** framing is the key differentiator: most competitors retrieve memories, but TriageTrace *validates and hardens* them. If we complete the submission artifacts (Alibaba Cloud deployment, 12+ scenarios, architecture diagram, 3-minute video) and keep the live evaluation numbers honest, we have a realistic path to a top-3 finish or track win.
